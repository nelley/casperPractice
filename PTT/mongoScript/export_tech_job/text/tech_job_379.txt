首先要感謝版上前輩的分享和指點 #1QJDqbKw
經過各方面調整後陸續接到一些面試的邀請
最終順利在年前從科技業取得正職
這份心得涵蓋轉換跑道和新手資料工程師的部分
希望我的經驗也能夠幫助到其他人  (文長請海涵)
--
我本來的專長是交通工程與數量統計分析
一年多前要脫離本行時完全沒有任何科技業相關的學歷證照
只有之前所存下足夠支撐生活一年的存款
還有因為興趣使然而具備的基礎程式寫作能力 (大概比 Hello world 好不了多少)

充分認知自己的不足之後下定決心拋開一切對薪水、頭銜的期待
在 29 歲高齡申請提供給大四和碩班學生的實習生職位
沒想到運氣很好的遇到一位好老闆
除了願意收我當學徒外也餵給我各種練功專案
於是前六個月在咬緊牙根支撐下
我從 MIS 的工作一邊學習大型企業 Infra 架構的基本
一邊在不斷犯錯的同時開發 Infra 相關資料的 ETL 模式並嘗試分析資料
現在回想起來具有 Infra 的相關知識其實還滿有幫助的
對資料處理平台的架構設計中
成本與效能的取捨能夠拿捏得更好

身為實習生的好處是正職員工雖然不會跟你討論工作的細節和機密
但是請教技術訣竅時常常有意外的收穫 (威脅感低?)
靠著無止境的 Stack overflow, github, gitbook, goole
這段時間就像對著消防栓喝水一樣
儘管不可能全部喝下去細節撒了滿地都是
但是技術能力卻是很有感的在提升

轉瞬間六個月馬上就到期
由於我手上仍有開發中未完成的專案
在老闆的鼎力協助下得到了額外六個月的延長實習合約
老實說當時確實曾經懷疑過
就算再六個月過去以這種速度成長真的能有競爭正職的實力嗎?
最後憑著寧死也不想回到交通業的執念
說服自己再拿出六個月和最後的存款押上去

這時候的我已經能夠熟練運用 AWS 各項服務 (by AWS CLI)
編寫 python script 來協助日常的 cloud admin工作或處理資料
以及獨力建構佈署 Hadoop ecosystem, Spark 來處理巨量資料
在工作漸趨穩定時天真的想著我是不是開始上軌道了
直到在一次跟老闆一起向 CIO 報告的機會後
CIO 建議我們嘗試協助其他產品團隊改進 data &amp; insight

這時候我才發現
原來資料常常會是以紊亂的形式塞在某個 SQL 伺服器的幾十個 table 裡面
做資料處理的工作確實實力很重要
但溝通與解釋的能力有可能更重要
很多時候對方聽說有資料專家來幫忙了
就興高采烈的丟一個資料庫給你然後期待你生一個 ML 模型出來自動飛天遁地
如何在不冒犯同事的情況下讓他理解巧婦難為無米之炊的現實
或是他想要的成果和給你的資料間有多大的落差
還有你需要他提供哪些必要的協助來實現他的願望
我在這個階段除了惡補 SQL 資料庫的相關技能
其餘精力都消耗在協調會議上

到了今年初知道大限已到
實習生的工作展延一次已是上限
在不確定能轉任正職的情況下重新開始修編履歷找工作
起初一天投十幾個職缺但是一週過去投了上百個音訊全無
在慌亂中到本版來求救
同時也想辦法在 LinkedIn, glassdoor 上面找建議來看
經過多次編輯修正終於開始有面試邀約

其中有參加過聯詠的 MIS 工程師 https://tinyurl.com/yarqw2j3 電話面試
由三位職缺主管主持
面試內容主要都是針對相關職務經歷與實作經驗
沒有刻意刁難或刺激情緒的問題
我覺得電話面試過程相當輕鬆愉快
也順利取得 F2F 面試的機會

最後在前往聯詠二面的前兩天
就在我剛要把面談人事資料表寄出去前
竟然接到原任職公司轉任正職的 offer letter
年底薪 N*14 month &gt; NTD $800K 績效獎金另計
因為事先有跟老闆約定如果能轉任正職會留下來繼續完成手上的專案
所以終於是塵埃底定可以安心過年了

就這樣...半路出家到科技業的大冒險終於能喘一口氣
說實話這種心情有點像電動玩具終於完成教學任務
好不容易可以開始真正的挑戰的心情

我不敢說照著我所做的就會成功
畢竟回顧過去一年很多關鍵時刻根本就是運氣好
但是我想鼓勵跟我一樣想要賭上一切改變未來的人
我覺得這件事有幾個重點:
        1.準備: 如果沒有足夠支撐一年當學徒的存款我不可能
        2.動機: 對運輸業的職業環境的痛恨驅使我挺過來
        3.自知: 誠實面對自己的不足設法補救

對不起，浪費您的人生看這種文筆欠佳的長文
預祝您新年快樂
